Development Conventions

This document outlines the development conventions and standards used throughout the ts-playground project.

Code Style​

TypeScript​

  • Use TypeScript for all new code
  • Prefer type inference over explicit type annotations when types are obvious
  • Use interfaces over types for object structures
  • Use PascalCase for types and interfaces
  • Use camelCase for functions, variables, and properties

Naming​

  • Use descriptive, meaningful names
  • Follow the pattern verbNoun for function names (e.g., getUserById)
  • Use PascalCase for class names and interfaces
  • Use UPPER_SNAKE_CASE for constants
  • Use descriptive variable names over short abbreviations

File Organization​

  • Group related files in directories
  • Use index.ts files to export from directories
  • Separate type definitions when they're used across modules
  • Keep files focused on a single responsibility

Error Handling​

  • Use custom error classes for application-specific errors
  • Handle errors gracefully and provide meaningful error messages
  • Log errors appropriately for debugging
  • Avoid throwing generic error objects

Testing​

  • Write unit tests for all business logic
  • Use descriptive test names that explain the expected behavior
  • Follow the AAA pattern (Arrange, Act, Assert) in tests
  • Test both positive and negative cases

Documentation​

  • Document public APIs with JSDoc
  • Include examples in documentation when helpful
  • Use clear, concise language
  • Keep documentation up to date with code changes
